## 🏗️ Архитектурные решения

### Централизованное управление ботами
Вместо запуска отдельных PM2 процессов для каждого бота, все боты работают как инстансы внутри одного процесса Node.js:

```
┌─────────────────────────────────────┐
│   PM2: bananzabot (1 процесс)      │
│                                     │
│  ┌──────────────────────────────┐  │
│  │  Main Bot (конструктор)      │  │
│  └──────────────────────────────┘  │
│                                     │
│  ┌──────────────────────────────┐  │
│  │  BotInstanceManager          │  │
│  │                              │  │
│  │  • Bot 1 instance            │  │
│  │  • Bot 2 instance            │  │
│  │  • Bot 3 instance            │  │
│  │  • ...                       │  │
│  └──────────────────────────────┘  │
└─────────────────────────────────────┘
```

**Преимущества:**
- 💾 Экономия памяти (один процесс вместо N процессов)
- ⚡ Быстрый старт новых ботов (нет overhead PM2)
- 🔄 Простое восстановление при перезапуске
- 📊 Централизованный мониторинг

## 🔧 Технологический стек

- **Node.js** - основной runtime
- **node-telegram-bot-api** - работа с Telegram Bot API
- **axios** - HTTP запросы
- **OpenAI API** (GPT-4o-mini) - генерация промптов
- **PM2** - process manager
- **JSON** - хранилище данных

## 🎯 Основной workflow

```
1. Пользователь → /start → @bananza_bot
                ↓
2. Описывает свой бизнес
                ↓
3. OpenAI генерирует промпт
                ↓
4. Пользователь создает бота в @BotFather
                ↓
5. Отправляет токен → @bananza_bot
                ↓
6. Бот валидирует токен
                ↓
7. Сохраняет в базу данных
                ↓
8. BotInstanceManager создает инстанс
                ↓
9. ✅ Новый бот работает!
```

## 📈 Производительность

### Ожидаемые характеристики:
- **Использование памяти**: ~50-100 MB базовый процесс + ~20-30 MB на каждого бота
- **CPU**: Минимальная нагрузка в idle, пики при обработке сообщений
- **Время деплоя**: 2-3 секунды
- **Время генерации промпта**: 5-10 секунд

### Ограничения:
- Рекомендуется не более 50 одновременно активных ботов
- OpenAI rate limits
- Telegram Bot API rate limits (30 msg/sec)
