# 🍌 Bananzabot - Telegram Bot Constructor

Главный бот-конструктор для автоматического создания и деплоя Telegram ботов с автоворонками.

## 🎯 Возможности

- **Конструктор ботов**: Создавайте ботов через диалог, просто описав свой бизнес
- **Интерактивный AI-помощник**: Проводит через процесс создания, задает уточняющие вопросы и помогает сформулировать правильные требования
- **Автоматическая генерация промптов**: AI создает оптимальный промпт на основе интерактивного диалога
- **Система уведомлений**: Боты могут отправлять сообщения в личку пользователей и каналы администраторов
- **Inline-кнопки в ответах**: Боты могут добавлять Telegram-кнопки через текстовую команду `[BUTTONS]` с JSON
- **Мгновенный деплой**: Боты автоматически деплоятся и запускаются в одном процессе
- **Централизованное управление**: Все боты работают в рамках одного процесса - экономия памяти и ресурсов
- **Встроенная аналитика**: Команда /stats для мониторинга активных ботов

## 📋 Требования

- Node.js 14+
- npm
- Telegram Bot Token (от @BotFather)
- Hydra AI API Key (для чата)
- OpenAI API Key (только для распознавания аудио)

## 🚀 Установка

1. Клонируйте проект:
```bash
cd /root/aisell/bananzabot
```

2. Установите зависимости:
```bash
npm install
```

3. Создайте `.env` файл:
```bash
cp .env.example .env
```

4. Заполните `.env` файл:
```env
TELEGRAM_BOT_TOKEN=your_main_bot_token_here
HYDRA_API_KEY=your_hydra_key_here
HYDRA_BASE_URL=https://api.hydraai.ru/v1
OPENAI_API_KEY=your_openai_api_key_here
```

```bash
cd /root/aisell/bananzabot
npm test

```

Опционально (реальные Telegram e2e через Pyrogram):

```bash
cd /root/aisell/bananzabot

```

`test:pyrogram:create`:
- проверяет создание бота через `@bananza_bot` (невалидный токен -> валидный токен);
- затем проверяет уже созданного бота (ответ на `/start`, уведомление владельцу, reply flow).

`test:pyrogram:ready`:
- быстрый smoke-тест уже готового бота: ответ на `/start` и на обычное сообщение.

Перед `test:pyrogram:create` задайте валидный токен через env или флаг:

```bash
cd /root/aisell/bananzabot
BANANZABOT_E2E_VALID_TOKEN="123456:ABCDEF..." npm run test:pyrogram:create
# или
python3 tests/e2e_pyrogram_test.py --valid-token "123456:ABCDEF..."
```

Для быстрого теста готового бота можно указать username:

```bash
cd /root/aisell/bananzabot
python3 tests/e2e_pyrogram_ready_bot_test.py --bot sometest33bot
```

## 💬 Использование

### Создание нового бота:

1. Откройте бота в Telegram
2. Отправьте `/start`
3. Опишите идею для бота (например: "Хочу бота для записи в салон красоты")
4. **Интерактивный диалог**: AI-помощник задаст уточняющие вопросы:
   - Какие услуги/продукты вы предоставляете?
   - Какую информацию нужно собирать?
   - Нужны ли уведомления в личку или канал?
5. После сбора всей информации AI сгенерирует оптимальный промпт
6. Протестируйте бота бесплатно по предоставленной ссылке
7. Если все устраивает - создайте токен через @BotFather и отправьте его
8. Готово! Бот автоматически задеплоится и начнет работать

Важно:
- Одно сообщение с описанием должно быть до `3500` символов (лимит Telegram), иначе бот попросит сократить текст и отправить одним сообщением.
- Если отправить API токен, который уже есть в системе, будет обновлен существующий бот с этим токеном (без создания дубликатов записей).

Примечание по `@bananzatestbot`: в тестовом диалоге в промпт прокидываются имя и `@username` тестирующего пользователя (если доступны), чтобы бот не отвечал шаблонным обращением вроде "Имя".

### Преимущества интерактивного подхода

- **Реалистичные ожидания**: AI объясняет ограничения (нет внешних API) и предлагает альтернативы
- **Более точные промпты**: Уточняющие вопросы помогают создать именно то, что нужно
- **Обучение**: Клиенты лучше понимают возможности AI-ботов

### Команды главного бота (@bananza_bot):

- `/start` - Начать создание нового бота
- `/mybots` - Посмотреть список своих ботов
- `/editbot` - Редактировать промпт существующего бота
- `/stopbot` - Остановить работающего бота
- `/startbot` - Запустить остановленного бота
- `/stats` - Статистика системы
- `/analytics` - Детальная аналитика активности
- `/cancel` - Отменить текущий процесс

### Команды в созданных ботах (для пользователей):

- `/start` - Начать диалог
- `/help` - Справка по командам
- `/clear` - Очистить историю диалога

Формат кнопок в ответах созданных ботов:
- После обычного текста бот может добавить блок:
`[BUTTONS]`
`[[{"text":"Каталог","callback_data":"catalog"}],[{"text":"Менеджер","url":"https://t.me/example"}]]`
- Поддерживаются inline-кнопки Telegram (`url`, `callback_data`, `switch_inline_query`, `switch_inline_query_current_chat`, `web_app`, `pay`).

### Команды в созданных ботах (только для владельца):

- `/mystats` - Персональная статистика бота
- `/users` - Список пользователей бота
- `/broadcast <текст>` - Рассылка по всем пользователям

## 🔄 Процессы

**Ключевая особенность**: Все боты работают в рамках одного процесса Node.js!

- Главный бот (bananzabot.js) - конструктор ботов
- Все созданные боты запускаются как инстансы внутри того же процесса
- При перезапуске все активные боты автоматически восстанавливаются
- Экономия памяти и процессорного времени

## 🌐 Веб-админка диалогов

- URL: `http://localhost:3182/admin`
- Домен: `http://bananzabot.wpmix.net/admin`
- CRM-фоллоуапы: `http://localhost:3182/admin/crm`
- Тестирование: `http://bananzabot.wpmix.net/admin/testing`
- Доступ только с IP: `212.193.45.174`, `89.185.84.184`, `80.211.131.142` (для остальных выводится `доступ по ip заперщен. ваш ip: ...`)
- Запуск через PM2: `pm2 start ecosystem.config.js`
- История в админке берется из `user_data/conversations/`. Ответы бота будут видны только если они логируются через `sendAndLogBotMessage()` или `conversationManager.addMessage(userId, 'assistant', ...)`.
- В админке можно выбрать модели: отдельно для генерации промптов и отдельно для клиентских ботов
- В админке есть форма активации Hydra refill-кода
- В админке есть кнопка `Check API key` (проверка Hydra API ключа из `.env` или вручную введенного ключа)
- В CRM есть процесс для менеджера: генерация follow-up через AI, отправка сообщения пользователю и фиксация статуса лида.
- `Generate AI follow-up` теперь редиректит на карточку лида (`/admin/authors/:userId`), где есть блок `CRM Follow-up` с генерацией, редактированием и кнопкой отправки.
- Follow-up ограничивается коротким форматом (до ~420 символов) и всегда содержит ссылку на `https://t.me/bananza_bot`.
- После получения API-токена нового клиентского бота автоматически генерируются `Name`, `About`, `Description`, команды, а также текстовые подсказки `Botpic` и `Description picture`; через Telegram Bot API применяются `setMyName`, `setMyShortDescription`, `setMyDescription`, `setMyCommands` (если API вернул ошибку, в чат уходит статус частичного применения).
- Генерация follow-up учитывает наличие `username`: если username известен, текст генерируется как личное сообщение от создателя; если username отсутствует — текст генерируется от имени `@bananza_bot` с фокусом на возврат пользователя в диалог бота.
- Отправка follow-up из CRM также учитывает это правило: при наличии username отправка идет из личного аккаунта менеджера, без username — через Bot API от `@bananza_bot` по `user_id`.
- После отправки CRM пытается добавить контакт в Telegram-папку `bananza` через `scripts/telegram_add_to_folder.py`.
- Для добавления в папку Telegram нужен валидный peer (id + access_hash). Если был только `user_id` и чат ещё не "известен" сессии, Telegram может вернуть `PEER_ID_INVALID`; скрипт `telegram_add_to_folder.py` делает дополнительный резолв через `get_users/get_dialogs`, но при полностью незнакомом контакте сначала нужен любой предыдущий контакт с пользователем.
- После отправки follow-up в карточку автора добавляется запись `[CRM follow-up sent ...]` в историю диалога.

## 📈 Tracking пользователей и Referral система

### Параметры запуска (Start parameters)

Бот поддерживает tracking источников через start параметры в формате `t_*`:

**Формат ссылки:**
```
https://t.me/bananza_bot?start=t_channelname
```

**Пример:**
```
https://t.me/bananza_bot?start=t_neirosetblog
```

### Что происходит при переходе по такой ссылке:

1. **Парсинг параметра** - бот распознает что это реферал из Telegram канала
2. **Уведомление админу** - отправляется уведомление с информацией:
   - User ID пользователя
   - Username (@username)
   - Источник: Telegram канал @channelname
3. **Сохранение в профиль** - в профиль пользователя добавляются поля:
   ```json
   {
     "referralSource": "telegram_channel:channelname",
     "referralParam": "t_channelname",
     "referralDate": "2025-12-18T..."
   }
   ```

### Защита данных при перезапуске

- **Профили не перезаписываются** - при повторном /start существующий профиль сохраняется
- **Referral данные защищены** - источник сохраняется даже при очистке истории диалога
- **Работает для созданных ботов** - все созданные боты также получают эту функциональность

### Пример использования:

```bash
# Создать реферальную ссылку для канала
echo "https://t.me/bananza_bot?start=t_mychannel"

# Результат в уведомлении админу:
# "Новый пользователь нажал старт в bananzabot:
#  User ID: 12345678,
#  Имя: Иван Иванов,
#  Username: @ivan123,
#  Источник: Telegram канал @mychannel"
```

## 📬 Система уведомлений

Созданные боты могут отправлять уведомления двух типов:

### 1. Личные сообщения пользователю (DM)
Используется для отправки подтверждений, деталей заявки, напоминаний.

**Пример использования:**
```
Пользователь: "Хочу записаться на маникюр 25 декабря"
Бот в чате: "Заявка принята! Подробности отправил в личные сообщения."
[NOTIFY_USER] ✅ Ваша заявка на маникюр 25 декабря принята. Мы свяжемся с вами в течение часа.
```

**Ограничения:**
- Пользователь должен сначала написать боту в личку (нажать START)
- Если пользователь не писал боту, уведомление не будет отправлено
- Бот уведомит об этом в групповом чате

### 2. Уведомления в канал администраторов
Используется для информирования администраторов о новых заявках/событиях.

**Пример использования:**
```
[NOTIFY_ADMIN] 📝 Новая заявка #123
От: @username
Услуга: Маникюр
Дата: 25.12.2024
Телефон: +7...
```

### Настройка уведомлений

При создании бота в интерактивном диалоге AI-помощник спросит:
- Нужно ли отправлять подтверждения клиентам в личку?
- Куда отправлять уведомления администратору - в канал или группу?

AI автоматически добавит в промпт инструкции по использованию команд `[NOTIFY_USER]` и `[NOTIFY_ADMIN]`.
Если канал/группа не указаны, уведомления администратору отправляются в личку создателю бота (user_id).

### Синтаксис команд в промпте

```
Когда пользователь оставляет заявку:
1. Подтверди приём в чате
2. Используй [NOTIFY_USER] для отправки деталей в личку
3. Используй [NOTIFY_ADMIN] для уведомления администраторов

Пример:
"Заявка принята!

[NOTIFY_USER] Ваша заявка принята...
[NOTIFY_ADMIN] Новая заявка от @user..."
```

## 🔄 Управление метаданными бота

**Важно**: Автоматическое обновление метаданных главного бота (bananzabot) **отключено**.

Метаданные бота (Name, Description, Commands) управляются **вручную** через @BotFather:
- `/setdescription` - установить полное описание
- `/setabouttext` - установить короткое описание
- `/setcommands` - настроить команды
- `/setuserpic` - установить аватар

Это позволяет контролировать профиль бота и избежать случайной перезаписи настроек при перезапуске.

## 🐛 Troubleshooting

### Бот не отвечает:
```bash
pm2 restart bananzabot
pm2 logs bananzabot --lines 50
```

### Проверить активные боты:
Отправьте `/stats` в главного бота

### Очистить все данные:
```bash
rm -rf bots_database/bots.json user_data/*
pm2 restart bananzabot
```

### 409 Conflict ошибки при перезапуске

Если в логах появляются ошибки `409 Conflict: terminated by other getUpdates request`:

**Причина**: При pm2 restart старый процесс не успевал остановить все инстансы созданных ботов до запуска нового процесса.

**Решение** (уже исправлено в коде):
- `ecosystem.config.js` с `kill_timeout: 10000` дает время на graceful shutdown
- В `bananzabot.js` добавлен `gracefulShutdown()` который останавливает все боты перед выходом

**Правильный запуск:**
```bash
pm2 delete bananzabot 2>/dev/null
pm2 start ecosystem.config.js
pm2 save
```
