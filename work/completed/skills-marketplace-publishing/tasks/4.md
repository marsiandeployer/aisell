**Files:**

Текущее состояние (строки 355–365):
```
// CHANGE: Product-aware CLAUDE.md template resolution
// WHY: Each product (simple_dashboard, simple_site) has its own client template
// REF: Templates live in products/{product_type}/CLAUDE.md.template
function getClaudeMdTemplatePath(): string {
  const productType = (process.env.PRODUCT_TYPE || '').toLowerCase();
  if (productType) {
    const productPath = path.join(__dirname, `../../products/${productType}/CLAUDE.md.template`);
    if (fs.existsSync(productPath)) return productPath;
  }
  return CLAUDE_MD_TEMPLATE_PATH;
}
```

После изменения:
- Строка 357 (REF-комментарий): `// REF: Templates live in products/{product_type}/SKILL.md`
- Строка 361 (path.join): суффикс `CLAUDE.md.template` → `SKILL.md`

**Dependencies:**

Задачи 1 (SKILL.md для simple_dashboard) и 3 (SKILL.md для simple_site) должны быть завершены до деплоя — если `SKILL.md` не существует на диске, функция вернёт CLAUDE_MD_TEMPLATE_PATH (фолбэк). Но в wave 2 задачи 3 и 4 могут выполняться параллельно, поскольку их нет между собой зависимостей по коду.

**Edge cases:**

- `CLAUDE_MD_TEMPLATE_PATH` (строка 352) использует `CLAUDE.md.example` — совсем другое имя, это фолбэк для случая когда PRODUCT_TYPE не задан или файл продукта не найден. Не трогать.
- Если `SKILL.md` для конкретного продукта не существует (например, `simple_crypto`), функция корректно фолбэчит на `CLAUDE_MD_TEMPLATE_PATH` — логика `fs.existsSync` уже обрабатывает этот случай.
- `buildClaudeMdContent()` не содержит `{{PROJECT_IDEA}}`-подстановки, если маркер отсутствует в файле — копирует content as-is без ошибок (confirmed per tech-spec Risks section).

**Implementation hints:**

Это строго хирургическое изменение. Менять нужно только 2 места: REF-комментарий и один string literal в path.join. Не трогать `buildClaudeMdContent()`, не трогать `CLAUDE_MD_TEMPLATE_PATH`.
