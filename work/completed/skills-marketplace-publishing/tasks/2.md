## Description

Create `products/simple_dashboard/marketplace.json` — a public metadata file for SkillsMP and GitHub marketplace indexing. This is a new file that does not exist yet.

The file provides structured metadata that external marketplace tools (skillsmp.com, GitHub marketplace scrapers) read to display a skill card for SimpleDashboard. The file must be valid JSON, contain the exact schema defined in tech-spec, and include the required tags so the skill is discoverable under dashboard, analytics, business, chartjs, tailwind, visualization, and spa categories.

This task is purely file creation — no code changes, no dependencies on other tasks. It runs in Wave 1 in parallel with Task 1 (SKILL.md for simple_dashboard).

## Details

**Files:**
- `products/simple_dashboard/marketplace.json` — new file, create from scratch using the exact schema from tech-spec Data Models section.

**Exact schema to use (from tech-spec):**
```json
{
  "name": "SimpleDashboard",
  "version": "1.0.0",
  "description": "Build professional business analytics dashboards as single-file index.html. Includes 7+ industry templates (e-commerce, SaaS, agency, restaurant, real estate), Chart.js charts, bilingual EN/RU support, adaptive business interview.",
  "author": "Noxon Digital Factory",
  "homepage": "https://simpledashboard.wpmix.net",
  "tags": ["dashboard", "analytics", "business", "chartjs", "tailwind", "visualization", "spa", "i18n"],
  "install": "Copy products/simple_dashboard/ to ~/.claude/skills/simpledashboard/",
  "screenshots": [
    { "path": "showcases/construction-crm/screenshot-1280x800.png", "caption": "Construction CRM Dashboard" },
    { "path": "showcases/sales-analytics-utm/screenshot-1280x800.png", "caption": "Sales Analytics + UTM" },
    { "path": "showcases/funnel-analytics/screenshot-1280x800.png", "caption": "Full Funnel Analytics" }
  ]
}
```

**Context on structure:**
- `product.yaml` has a `chrome_store` section with `name`, `short_description`, `keywords`, `screenshots` — use it as a structural model to understand how the product metadata is organized, but `marketplace.json` is its own separate file with its own schema.
- `seo.keywords` in `product.yaml` (primary: "бизнес дашборд", "аналитика продаж", "KPI дашборд", "dashboard builder"; secondary: "визуализация данных", etc.) informed the tech-spec tag choices — the tags in marketplace.json are the English equivalents already determined.
- The showcase directories `construction-crm`, `sales-analytics-utm`, and `funnel-analytics` all exist under `products/simple_dashboard/showcases/` — confirmed by filesystem check.

**Decision context (from tech-spec Decision 3):**
`marketplace.json` is a separate public file, not a new section in `product.yaml`. `product.yaml` contains internal config (bot tokens via env vars, internal URLs, system prompts) and must not be published. `marketplace.json` is a public-facing artifact designed for external tools to read.

**Dependencies:** None — this task has no depends_on and runs independently in Wave 1.

**Edge cases:**
- The screenshot `.png` files may not exist yet (screenshot generation is a separate process), but the paths in `marketplace.json` should still point to the spec-defined locations — the JSON file does not require the PNGs to exist for the verification to pass.
- Do not add any fields not in the schema — keep the file minimal and exact.
