# Task 3: webchat.ts foundation — invite storage + ethers + rate limiters + fail-fast

## Required Skills

Before starting, load: `/skill:code-writing` — [SKILL.md](/root/.claude/skills/code-writing/SKILL.md)

## Description

This task lays the infrastructure groundwork in `webchat.ts` that all subsequent invite and OAuth tasks depend on. It does not add any HTTP endpoints — only types, constants, helpers, and startup wiring.

Specifically, the task adds:
- `import { ethers } from 'ethers'` (currently only in auth-api.ts, not in webchat.ts)
- `InviteRecord` type and `INVITES_PATH` constant for disk-persisted invite storage
- `readInvites` / `writeInvites` helpers following the existing `readSessions` / `writeSessions` pattern
- `inviteTokens` Map populated from `invites.json` at startup (analogous to how sessions are loaded)
- Two new `SlidingWindowRateLimiter` instances: `rlInviteUser1h` (20/hour/userId) and `rlOAuthCallbackIp10m` (20/10min/IP), both added to `allLimiters` so they participate in the periodic sweep
- `signChallenge(privateKey, dashboardId)` helper that builds a challenge JSON string, signs it via `ethers.Wallet`, and returns `{ challenge: string, signature: string }`
- Extended `MagicLinkEntry` type adding optional `dashboardJwt?: string` field (currently the Map stores `{ userId: string; expires: number }` inline — this task adds the named type)

Additionally, two pre-existing security fallbacks must be removed and replaced with fail-fast startup assertions:
- Line 3811: `process.env.JWT_SECRET || 'magic-secret'` — remove the `|| 'magic-secret'` fallback
- Line 5099: `process.env.INTERNAL_API_KEY || ''` — remove the `|| ''` fallback
- Add `if (!process.env.JWT_SECRET) throw new Error('JWT_SECRET required')` at startup (in the `main()` function, before `const app = express()`, following the same pattern already used in auth-api.ts lines 21-25)
- Add the same assertion for `INTERNAL_API_KEY`

This task is in Wave 2 alongside Task 4 (auth-api.ts) — they modify different files and can run in parallel.
