# Task 1: GOOGLE_CLIENT_SECRET env setup

## Required Skills

Before starting, load:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](/root/.claude/skills/code-writing/SKILL.md)

## Description

The Google OAuth authorization-code flow (Task 6) requires `GOOGLE_CLIENT_SECRET` to exchange the `code` parameter returned by Google for an ID token. Without this secret, the `OAuth2Client.getToken(code)` call will fail.

Currently, `botplatform/.env.auth` has `JWT_SECRET`, `INTERNAL_API_KEY`, `PG_PASSWORD`, and `GOOGLE_AUTH_TEST_SECRET`, but no `GOOGLE_CLIENT_SECRET`. This task adds that missing value.

In addition, `https://simpledashboard.wpmix.net/api/auth/google-dashboard-callback` must be registered as an authorized redirect URI in the Google Cloud Console for the existing OAuth client (`531979133429-b20qi1v15bgoq724tfk808lr1u3a1ev2`). Without this registration, Google will reject the authorization-code redirect and the entire guest OAuth flow cannot work.

This task is a prerequisite for all OAuth callback work (Tasks 6+). No code changes to TypeScript files are needed — only the `.env.auth` file and the Google Cloud Console configuration.

## What to do

1. Retrieve the `GOOGLE_CLIENT_SECRET` value for OAuth client `531979133429-b20qi1v15bgoq724tfk808lr1u3a1ev2` from Google Cloud Console (project: the one owning `simpledashboard.wpmix.net`).

2. Append `GOOGLE_CLIENT_SECRET=<value>` to `/root/aisell/botplatform/.env.auth`. Keep the file mode at 600. Do not commit this file to git (it is already in `.gitignore`).

3. In Google Cloud Console, navigate to APIs & Services → Credentials → the OAuth 2.0 client `531979133429-b20qi1v15bgoq724tfk808lr1u3a1ev2` → Edit → Authorized redirect URIs. Add `https://simpledashboard.wpmix.net/api/auth/google-dashboard-callback` and save.

4. Verify that the `simpledashboard-web` PM2 process picks up the new env var. The process reads `.env.auth` at startup via `start-webchat-simpledashboard.sh`, so a restart is needed after the file is updated.

## Details

**Files:**
- `/root/aisell/botplatform/.env.auth` — append one line: `GOOGLE_CLIENT_SECRET=<value>`. Current contents: `JWT_SECRET`, `INTERNAL_API_KEY`, `PG_PASSWORD`, `GOOGLE_AUTH_TEST_SECRET`. Do not remove or modify existing values.

**Google Cloud Console context:**
- OAuth client ID: `531979133429-b20qi1v15bgoq724tfk808lr1u3a1ev2`
- Redirect URI to add: `https://simpledashboard.wpmix.net/api/auth/google-dashboard-callback`
- This is the only redirect URI needed for the multi-user-auth feature
- The `GOOGLE_CLIENT_ID` (`531979133429-b20qi1v15bgoq724tfk808lr1u3a1ev2`) is already referenced in `webchat.ts` via `process.env.GOOGLE_CLIENT_ID` — that env var should already be set (from the existing Google auth flow). This task only adds the *secret*. Note: `GOOGLE_CLIENT_SECRET` is NOT yet referenced in webchat.ts — Task 6 will add its usage in the OAuth callback handler.

**How `.env.auth` is loaded:**
`start-webchat-simpledashboard.sh` uses `set -a; source /root/aisell/botplatform/.env.auth; set +a` which exports all variables into the shell environment before PM2 starts the process. Restarting via `pm2 restart simpledashboard-web` alone is not enough if PM2 restarts with cached env — the process must be stopped and started via the launch script, or restarted with `--update-env` if the env is already in the PM2 config.

**Edge cases:**
- If `GOOGLE_CLIENT_SECRET` already exists in `.env.auth` with a different (test) value, overwrite it — do not create a duplicate key. Use a targeted append only after verifying the key is absent: `grep -q GOOGLE_CLIENT_SECRET .env.auth || echo "GOOGLE_CLIENT_SECRET=..." >> .env.auth`.
- The `.env.auth` file is in `.gitignore`; verify with `git check-ignore` before trusting the pattern.
- Google Cloud Console may require a few minutes for redirect URI changes to propagate.

**Security note:**
The `GOOGLE_CLIENT_SECRET` must never appear in log output, process arguments, or HTTP responses. The existing `webchat.ts` code reads it only as `process.env.GOOGLE_CLIENT_SECRET` at request time (it is not printed or exposed). The security-auditor reviewer should confirm this.
