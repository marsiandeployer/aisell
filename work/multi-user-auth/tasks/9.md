## Description

This task injects a `<script id="auth-widget-loader">` block into the HTML served from `d*.wpmix.net`, alongside the already-existing magic-link script that runs in the same `html.replace('</body>', ...)` call.

The widget is the client-side half of the guest auth flow. It is responsible for:
- Silently auto-authenticating returning guests via a CORS call to `GET /api/auth/invite/status` (implemented in Task 8).
- Showing a Google OAuth button to first-time guests who arrived via `?invite=TOKEN`.
- Handling `?error=no_access` and `?error=service_unavailable` redirects from the OAuth callback (Task 6).
- Doing nothing (exiting immediately) if `sessionStorage.dashboard_jwt` is already set.

The server also generates a per-request `window.__OAUTH_NONCE__` (16-byte hex from `crypto.randomBytes`) that is embedded as an inline JS variable in the same injected block. This nonce is simultaneously written into the `webchat_session` entry so that the OAuth callback (Task 6) can validate it and prevent CSRF.

The widget is injected only when `chatSettings.ownerAddress` is set. Dashboards without an `ownerAddress` are publicly accessible and must receive no widget.

The widget must not render any webchat-specific UI (no "Back to chat", no Profile link, no Logout link). Guests should see only the auth widget or nothing.

**What to change in the injection block:**


**Session nonce storage:** Sessions in webchat.ts are file-based (stored in `sessions.json` via `readSessions()`/`writeSessions()`), not an in-memory Map. To store the nonce: parse the `webchat_session` cookie value via `parseCookieHeader(req.headers.cookie || '')`, call `readSessions()` to get the current sessions array, find the matching session by `sessionId`, add `oauthNonce: nonce` to that session object, and call `writeSessions(sessions)` to persist. The `WebSession` type may need to be extended with `oauthNonce?: string` field. The nonce is later read by the OAuth callback handler (Task 6) to validate CSRF protection.

**CSP change (lines ~3839â€“3846):** Change:
```
"connect-src 'self'; "
```
to:
```
"connect-src 'self' https://simpledashboard.wpmix.net; "
```
`accounts.google.com` is navigated to (not fetched with `fetch()`), so it does not need `connect-src`.

**Overlay structure in the widget:** The widget script must inject an `authOverlay` div only if one is not already present in the document (check `!document.getElementById('authOverlay')`). The overlay structure must match what `CLAUDE.md.template` documents: `id="authOverlay"` container with children having `data-overlay="no-access"` and `data-overlay="service-unavailable"`. **Nonce is per-request:** A new nonce is generated on every HTML page load. This is correct â€” each page load may start a new OAuth flow. The previously stored nonce in the session is overwritten each time.
