## Description

Deploy the multi-user-auth feature (Guest Mode) to production. The feature spans changes across `botplatform/src/webchat.ts` and `botplatform/src/auth-api.ts` introduced in tasks 1–14. All automated tests pass after Task 14 (Pre-deploy QA). This task builds the compiled TypeScript output, reloads the running PM2 process, confirms the process is healthy, and verifies the feature with the manual smoke test checklist from user-spec "Ручная проверка".

The target process is `simpledashboard-web` (port 8094). Deployment is in-place on the current application server (95.217.227.164). No reverse proxy changes are needed — nginx config on 62.109.14.209 is unchanged.

## What to do

1. Build TypeScript: run `npm run build` in `/root/aisell/botplatform/`. The build must exit 0.
2. Reload PM2: `pm2 reload simpledashboard-web --update-env`. This replaces the running process gracefully with zero downtime.
3. Confirm process health: `pm2 status simpledashboard-web` shows `online`. Check `pm2 logs simpledashboard-web --lines 50 --nostream` for startup errors, unexpected exceptions, or missing env var assertions.
4. Run manual smoke test checklist (user-spec "Ручная проверка"):
   - Owner: open `/profile` → section "Поделиться дашбордом" is present → click "Создать invite-ссылку" → URL appears
   - Open the URL in incognito → dashboard is behind blur, auth widget with "Войти через Google" button is visible
   - Click "Войти через Google" → complete Google OAuth → return to dashboard → blur removed, widget hidden
   - Close and reopen `d{userId}.wpmix.net` → no widget shown (session alive, returning guest flow)
   - Owner: click "Отозвать ссылку" → open the old invite URL in a new incognito window → complete login → overlay "no-access" appears

## Details

**Deployment target:** Application server `95.217.227.164` (current server). All PM2 processes, code, and data live here. Reverse proxy `62.109.14.209` requires no changes.

**Process:** `simpledashboard-web` on port 8094. Started via `/root/aisell/botplatform/start-webchat-simpledashboard.sh`. Use `pm2 reload` (not `pm2 restart`) to avoid downtime.

**Build output:** `npm run build` in `botplatform/` compiles TypeScript to `dist/`. The PM2 process uses the compiled JS. Always rebuild before reload.

**Environment variables:** Required env vars (`JWT_SECRET`, `INTERNAL_API_KEY`, `GOOGLE_CLIENT_SECRET`) must be present before reload. They come from `botplatform/.env.auth` loaded by the start script. If the process exits immediately after reload, check these assertions added in Task 3.

**Log check pattern:** After reload, wait 5–10 seconds before reading logs. The process logs startup messages including `Webchat server started`. Absence of error lines confirms a clean start.

**`GOOGLE_CLIENT_SECRET` prerequisite:** Task 1 added this to `.env.auth`. Confirm it is non-empty: `grep GOOGLE_CLIENT_SECRET /root/aisell/botplatform/.env.auth`. If missing, the callback handler returns 500 and blocks the entire OAuth path.

**Manual smoke test — obtaining owner session:** Log into `https://simpledashboard.wpmix.net` via Google OAuth as the dashboard owner, then copy the `webchat_session` cookie value from browser DevTools to use in curl commands.

**invites.json:** After the first smoke-test invite generation, `data/webchat/invites.json` should exist at `/root/aisell/botplatform/data/webchat/invites.json`. Check it exists and is valid JSON.

**Rollback:** If `pm2 reload` fails or the process is unhealthy, `pm2 restart simpledashboard-web` with the previous build. In the worst case, revert the git commit and rebuild.

**Dependencies:** Task 14 (Pre-deploy QA) must be complete and all tests passing before starting this task.
