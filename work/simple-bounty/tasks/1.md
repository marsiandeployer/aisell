# Task 1: Product infrastructure

## Required Skills

Before starting, load:

## Description

Create the full infrastructure layer for the `simple_bounty` product so that `simplebounty-web` PM2 process starts and the webchat server responds on port 8097.

This task follows the same Product-as-Configuration model used by SimpleDashboard: a `products/simple_bounty/` folder contains product metadata, a dedicated bash launch script sets environment variables, a PM2 entry in `ecosystem.config.js` manages the process lifecycle, and an nginx vhost on VM104 proxies HTTP traffic to the local port.

No application logic is written in this task. The result is an empty (but functional) webchat endpoint at `http://localhost:8097` and `http://simplebounty.wpmix.net`, ready for Wave 2 tasks to add bounty-specific API routes.

## What to do

1. Create `products/simple_bounty/` directory with:
   - `product.yaml` — product metadata following the `simple_dashboard` structure: `product_id`, `name`, `tagline`, `description`, `version`, `status`, `telegram` section (port 8097, PRODUCT_TYPE simple_bounty), `visual` section
   - `CLAUDE.md.workspace` — empty placeholder file (will be filled in Task 6)

2. Create `botplatform/start-webchat-simplebounty.sh` — bash launch script modeled on `start-webchat-simpledashboard.sh` with these key differences:
   - `PRODUCT_TYPE=simple_bounty`
   - `WEBCHAT_PORT=8097`
   - `WEBCHAT_TITLE="SimpleBounty - AI Bounty Campaign Builder"`
   - `WEBCHAT_SUBTITLE="Create bounty campaigns with AI"`
   - `GOOGLE_OAUTH_REDIRECT_URI=https://simplebounty.wpmix.net/api/auth/google-dashboard-callback`
   - An appropriate `WEBCHAT_INIT_MESSAGE` in English and Russian describing the bounty platform
   - Make the script executable (`chmod +x`)

3. Add `simplebounty-web` entry to `botplatform/ecosystem.config.js` — copy the `simpledashboard-web` block and change: name, script path, log file names (use `simplebounty-web-error.log` / `simplebounty-web-out.log`)

4. Create `/etc/nginx/sites-available/simplebounty.wpmix.net` — nginx vhost config modeled on `/etc/nginx/sites-available/simpledashboard.wpmix.net`: `server_name simplebounty.wpmix.net`, `proxy_pass http://127.0.0.1:8097`, same proxy headers and timeout values. No showcases location block needed for this task.

5. Enable the vhost: `ln -sf /etc/nginx/sites-available/simplebounty.wpmix.net /etc/nginx/sites-enabled/simplebounty.wpmix.net`

6. Test nginx config: `nginx -t` and reload: `nginx -s reload`

7. Start the process: `pm2 start /root/aisell/botplatform/ecosystem.config.js --only simplebounty-web`

8. Save PM2 state: `pm2 save`

9. Verify: `pm2 status simplebounty-web` shows `online`; `curl -s -o /dev/null -w "%{http_code}" http://localhost:8097` returns `200`

## Details

**Files to create/modify:**

`products/simple_bounty/product.yaml` — new file. Model it on `products/simple_dashboard/product.yaml`. Key fields:
- `product_id: "simple_bounty"`
- `name: "SimpleBounty"`
- `tagline: "Launch bounty campaigns with AI. Create tasks, set rewards, reward participants."`
- Under `telegram:`: `webchat_url: "https://simplebounty.wpmix.net"`, `webchat_port: 8097`
- Under `visual:`: choose a distinct color (e.g. green `#10B981`) to differentiate from SimpleDashboard blue
- `status: "development"`

`products/simple_bounty/CLAUDE.md.workspace` — placeholder only. Task 6 fills the content. Can be an empty file or contain a single comment line.

`botplatform/start-webchat-simplebounty.sh` — new file. Key env vars:
```
PRODUCT_TYPE=simple_bounty
WEBCHAT_PORT=8097
GOOGLE_OAUTH_REDIRECT_URI=https://simplebounty.wpmix.net/api/auth/google-dashboard-callback
ENABLE_GOOGLE_AUTH=true
BOT_LANGUAGE=en
ENABLE_ONBOARDING=false
DISABLE_PAYMENT_FLOW=true
NOXONBOT_DISABLE_AUTO_START=true
DISABLE_START_NOTIFICATIONS=true
SKIP_GLOBAL_MESSAGE_HISTORY=true
WEBCHAT_INIT_WITH_START=false
```
The script must source `/root/aisell/noxonbot/.env` and `/root/aisell/botplatform/.env.auth` (same pattern as simpledashboard script). The final command is `cd /root/aisell/noxonbot && npm run webchat`.

`botplatform/ecosystem.config.js` — add new entry after `simplecrypto-web` block:
```js
// SimpleBounty webchat (port 8097, PRODUCT_TYPE=simple_bounty)
{
  name: 'simplebounty-web',
  cwd: '/root/aisell/botplatform',
  script: 'start-webchat-simplebounty.sh',
  interpreter: 'bash',
  watch: false,
  max_restarts: 10,
  min_uptime: '15s',
  env: {
    PATH: '/root/.nvm/versions/node/v22.21.1/bin:/root/.local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin'
  },
  error_file: '/root/.pm2/logs/simplebounty-web-error.log',
  out_file: '/root/.pm2/logs/simplebounty-web-out.log',
  merge_logs: true,
  log_date_format: 'YYYY-MM-DD HH:mm:ss'
},
```

`/etc/nginx/sites-available/simplebounty.wpmix.net` — new file. Minimal config, proxy everything to port 8097:
```nginx
server {
    listen 80;
    server_name simplebounty.wpmix.net;

    location / {
        proxy_pass http://127.0.0.1:8097;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300;
        proxy_send_timeout 300;
        proxy_connect_timeout 60;
    }
}
```
No `/showcases/` block needed — that is a SimpleDashboard-specific static gallery pattern not needed for SimpleBounty in this task.

**Dependencies:**
- No dependency on other tasks (Wave 1)
- Requires `noxonbot/.env` to exist (it does — used by all other webchat products)
- Requires `botplatform/.env.auth` to exist (it does — used by simpledashboard)
- DNS for `simplebounty.wpmix.net` is handled in Task 14 (Deploy). For local testing, `curl http://localhost:8097` is sufficient — no DNS needed.

**Edge cases:**
- If `pm2 start --only simplebounty-web` fails: check that the script path in ecosystem.config.js matches the actual file at `/root/aisell/botplatform/start-webchat-simplebounty.sh` (not under `noxonbot/`)
- The `cwd` in the ecosystem entry must be `/root/aisell/botplatform` (not `noxonbot`), consistent with `simpledashboard-web` and `simplecrypto-web`
- The start script itself does `cd /root/aisell/noxonbot && npm run webchat` — the cwd change happens inside the script, not in ecosystem.config.js
- `chmod +x botplatform/start-webchat-simplebounty.sh` is required; PM2 with `interpreter: 'bash'` will execute it but the file should still be executable as a safeguard
- Do not add a `SIMPLEBOUNTY_BOT_TOKEN` env var in the start script — it is optional (Task 8 handles notifications); absence means notifications are silently skipped, which is the correct MVP behavior
- `WEBCHAT_INIT_WITH_START=false` — must be set, same as simpledashboard. Setting it to `true` causes a double-start message that breaks the bounty AI dialog flow

**Google OAuth note (for reference):**
The `GOOGLE_OAUTH_REDIRECT_URI` is set in the script so the webchat server can use it for the `/api/auth/google-dashboard-callback` handler. The actual Google Cloud Console registration of this redirect URI is part of Task 14 (Deploy), not this task. For local testing on port 8097, Google OAuth will not work until the redirect URI is registered — but the server itself will start and serve the webchat HTML without issue.
