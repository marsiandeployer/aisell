## Description

Создаём файл `products/simple_bounty/index.html.template` — статический SPA-шаблон страницы bounty-кампании для участников. Этот HTML-файл публикуется AI-агентом в `group_data/user_{creatorId}/index.html` при создании кампании и открывается участниками по адресу `d{creatorId}.wpmix.net`.

Страница выполняет следующие функции для участника:
1. Авторизация через Google OAuth с использованием Auth SDK (`window.SD`) из `simplebounty.wpmix.net/sdk/auth.js`
2. Показ списка заданий кампании с наградами в поинтах (публичный endpoint, без auth)
3. Форма сабмита доказательства: текст или URL с client-side URL-валидацией (только `http://` или `https://`)
4. Отображение статуса заявки участника (pending / approved / rejected) с lazy auto-approve
5. Лидерборд топ-10 участников по сумме поинтов
6. Creator-предупреждение: если эскроу кампании равен 0 — показывать предупреждение «Пополните эскроу»

Шаблон использует те же паттерны, что `simple_dashboard`: Tailwind CSS via CDN, тёмная цветовая схема, Auth SDK от `simplebounty.wpmix.net`. Стиль страницы (фон, акцентные цвета, заголовок) содержит placeholder-переменные для AI-генерации уникального визуала при создании кампании.

Все API-вызовы идут к `/api/bounty/*` — на том же домене `d{creatorId}.wpmix.net`, который через nginx wildcard проксируется в `simplebounty-web` (порт 8097). Авторизация на authenticated endpoints — через JWT из `sessionStorage` (заголовок `Authorization: Bearer {jwt}`), который Auth SDK кладёт после Google OAuth.

Зависит от Task 3 (Escrow API) — к моменту создания этого шаблона уже существуют: `/api/bounty/campaigns/:id/tasks` (GET, public), `/api/bounty/campaigns/:id/tasks/:taskId/submissions` (POST, auth), `/api/bounty/campaigns/:id/tasks/:taskId/submissions/my` (GET, auth), `/api/bounty/campaigns/:id/leaderboard` (GET, public), `/api/bounty/campaigns/:id/escrow/balance` (GET, creator).

Помимо HTML-шаблона, задача требует двух изменений в `botplatform/src/webchat.ts`:

- `getAuthSdkJs()` сейчас hardcode-ит `AUTH_API = 'https://simpledashboard.wpmix.net'` (line 611). Для `simplebounty-web` SDK должен указывать на `simplebounty.wpmix.net`. Нужно параметризовать функцию, чтобы она принимала `authApiBaseUrl` и подставляла его вместо hardcoded значения.
- Wildcard-обработчик `d{N}.wpmix.net` (lines 4529-4537) выставляет CSP `script-src`, содержащий только `simpledashboard.wpmix.net`. Загрузка скрипта `simplebounty.wpmix.net/sdk/auth.js` будет заблокирована браузером для participant pages без добавления `simplebounty.wpmix.net` в этот список.

**Как устроен роутинг для participant page:**

Nginx wildcard `*.wpmix.net` → `simplebounty-web` (порт 8097). Сервер в `webchat.ts` определяет userId из hostname (`d{N}.wpmix.net` → userId=N), читает `group_data/user_{N}/index.html` и отдаёт его напрямую (статические файлы через `res.sendFile`). Auth-защита на уровне nginx отсутствует — вся авторизация в браузере через SDK.

**Auth SDK для simplebounty:**

Auth SDK подключается строкой `<script src="https://simplebounty.wpmix.net/sdk/auth.js"></script>`. Это тот же `getAuthSdkJs()` из `webchat.ts` (line 600), но у `simplebounty-web` AUTH_API = `simplebounty.wpmix.net`. SDK автоматически:
- Показывает Google Sign-In overlay при открытии страницы
- После успешной авторизации кладёт JWT в `sessionStorage` (ключ `dashboard_jwt`)
- Диспатчит событие `sd:auth` в `document`
- Предоставляет `window.SD.getUser()` → `{name, email, address, dashboardId}`
- `SD.logout()` — выход

Для authenticated запросов к `/api/bounty/*` нужно добавить заголовок `Authorization: Bearer {jwt}` из `sessionStorage.getItem('dashboard_jwt')`.

**API endpoints (все на том же домене d{N}.wpmix.net):**

- `GET /api/bounty/campaigns/{campaignId}/tasks` — публичный, без JWT. Возвращает массив `Task[]`
- `POST /api/bounty/campaigns/{campaignId}/tasks/{taskId}/submissions` — требует JWT. Body: `{proof: string}`. Ответ 201 `{id, status: 'pending', ...}` или 409 если дубликат
- `GET /api/bounty/campaigns/{campaignId}/tasks/{taskId}/submissions/my` — требует JWT. Ответ: `{status, pointsAwarded?, submittedAt}` или 404 если нет заявки
- `GET /api/bounty/campaigns/{campaignId}/leaderboard` — публичный. Ответ: `{entries: [{participantId, participantName, totalPoints}]}`
- `GET /api/bounty/campaigns/{campaignId}/escrow/balance` — публичный (GET только). Ответ: `{balance: number}`

**Decision 8 — OAuth redirect_uri:**

`start-webchat-simplebounty.sh` задаёт `GOOGLE_OAUTH_REDIRECT_URI=https://simplebounty.wpmix.net/api/auth/google-dashboard-callback`. В шаблоне ничего дополнительно не нужно — Auth SDK сам вызывает `/api/auth/config` для получения `oauthCallbackUrl`.

- Сеть недоступна при загрузке tasks / leaderboard: показать inline сообщение об ошибке, не crash
- Proof — пустая строка: валидация на клиенте, не отправлять пустой сабмит
- Proof — выглядит как URL без схемы (напр. `example.com`): проверить что не начинается с `http://` или `https://` → показать inline ошибку «URL must start with http:// or https://»
- Несколько заданий в кампании: список, каждое со своей формой и статусом
- Уже есть заявка по данному заданию: `GET /submissions/my` вернёт статус, форму скрыть, показать статус
