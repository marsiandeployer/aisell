# Task 6: AI webchat prompt (SKILL.md)

## Required Skills

Before starting, load: `/skill:prompt-master` — [SKILL.md](/root/.claude/skills/prompt-master/SKILL.md)

## Description

This task creates the AI system prompt for the Simple Bounty webchat product. The AI acts as a campaign creation assistant — it guides the creator through naming a campaign, writing a description, adding tasks with point rewards, funding the escrow, and publishing.

Three deliverables:
1. `products/simple_bounty/SKILL.md` — the AI system prompt that drives the webchat dialog. This file is read by the bot on startup (via `getClaudeMdTemplatePath`) and injected as the Claude system prompt for every new webchat session under `PRODUCT_TYPE=simple_bounty`. The prompt must teach Claude how to call the bounty API endpoints directly (POST `/api/bounty/campaigns`, POST `/api/bounty/campaigns/:id/tasks`, POST `/api/bounty/campaigns/:id/escrow/deposit`, POST `/api/bounty/campaigns/:id/publish`) and guide the creator step by step.
2. `products/simple_bounty/CLAUDE.md.workspace` — the instructions visible to campaign participants (not creators). This file is copied into the user workspace and describes what the participant can do on `d{creatorId}.wpmix.net`.
3. `botplatform/src/bot.ts` — add a `PRODUCT_TYPE=simple_bounty` branch in the `handleStart` routing block (lines 1247–1262), so the bot sends an appropriate welcome message when a creator opens the Telegram bot for this product.

The pattern to follow is `products/simple_dashboard/SKILL.md` — a lengthy system prompt that defines the AI persona, lists available API calls, and provides guardrails. The `getClaudeMdTemplatePath` function in `bot.ts` (line 388) already auto-resolves to `products/{productType}/SKILL.md` by `PRODUCT_TYPE` env var, so no routing change is needed for the prompt itself — only for the bot welcome message.

## What to do

1. Create `products/simple_bounty/SKILL.md` with the full AI system prompt for the bounty campaign creator workflow. The prompt must cover:
   - AI persona: friendly campaign creation assistant for Simple Bounty
   - Creator workflow sequence: ask for campaign name → ask for description → create campaign via API → add tasks (loop: task title, description, reward in points) → guide escrow deposit → publish campaign → deliver the participant link `d{creatorId}.wpmix.net`
   - API calls the AI must make (with exact endpoint paths and expected request body shapes from tech-spec Data Models)
   - How to handle API errors in dialog (e.g., escrow insufficient for deposit of 0, validation errors)
   - Guardrails: do not expose internal token/session, do not modify other users' campaigns, do not skip escrow before publish
   - Brief explanation of the participant side for the creator's benefit

2. Create `products/simple_bounty/CLAUDE.md.workspace` with participant-facing instructions: what `d{creatorId}.wpmix.net` contains, how to submit proof (text or URL), what statuses mean (pending/approved/rejected), how auto-approve works (48 hours), and what the leaderboard shows.

3. In `botplatform/src/bot.ts`, add a new branch after the `simple_dashboard` block (after line 1262) that handles `productType === 'simple_bounty'` and sends an appropriate start message. Use the same pattern as the `simple_site` branch (line 1248–1250): `await this.replyTr(ctx, 'start.simple_bounty')`. Add the corresponding translation key `start.simple_bounty` to the i18n translation files if they exist, or include a fallback inline reply.
