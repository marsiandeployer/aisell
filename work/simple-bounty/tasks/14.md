## Description

Деплой продукта simplebounty в production. Задача охватывает весь деплой-чеклист: запуск PM2 процесса `simplebounty-web` на app server (95.217.227.164), DNS-запись `simplebounty.wpmix.net → 62.109.14.209` через Cloudflare API, активация nginx vhost на VM104 (конфиг создан в Task 1 — нужна симлинка в sites-enabled + reload), создание nginx vhost и SSL через certbot на reverse proxy 62.109.14.209, и регистрация Google OAuth redirect URI в Google Cloud Console (Decision 8).

Запускается параллельно с Task 13 (Pre-deploy QA), оба зависят от Task 12.

После выполнения `https://simplebounty.wpmix.net` отвечает 200 и Google OAuth корректно работает для участников bounty-кампаний.

## What to do

1. Запустить PM2 процесс `simplebounty-web` на VM104 (95.217.227.164):
   - `cd /root/aisell/botplatform && pm2 start ecosystem.config.js --only simplebounty-web`
   - Убедиться: `pm2 status simplebounty-web` → online; `curl http://localhost:8097` → 200

2. Добавить DNS-запись `simplebounty.wpmix.net → 62.109.14.209` через Cloudflare:
   - `node /root/space2/cloudflare_subdomain_manager.js add simplebounty.wpmix.net 62.109.14.209`
   - Проверить: `node /root/space2/cloudflare_subdomain_manager.js list | grep simplebounty`

3. Активировать nginx vhost на VM104:
   - Создать симлинку: `ln -s /etc/nginx/sites-available/simplebounty.wpmix.net /etc/nginx/sites-enabled/simplebounty.wpmix.net`
   - Если файл sites-available не существует (Task 1 не создал) — создать по образцу `/etc/nginx/sites-available/simpledashboard.wpmix.net`, изменив `server_name` и `proxy_pass` на 8097
   - `nginx -t && nginx -s reload`

4. Создать nginx vhost на reverse proxy 62.109.14.209:
   - `ssh -i ~/.ssh/github_deploy_key root@62.109.14.209`
   - Создать `/etc/nginx/sites-available/simplebounty.wpmix.net` (see Details для шаблона)
   - `ln -s /etc/nginx/sites-available/simplebounty.wpmix.net /etc/nginx/sites-enabled/simplebounty.wpmix.net`
   - `nginx -t && nginx -s reload`

5. Выпустить SSL сертификат на reverse proxy:
   - `certbot --nginx -d simplebounty.wpmix.net --non-interactive --agree-tos -m admin@wpmix.net`
   - `nginx -t && nginx -s reload`

6. Зарегистрировать Google OAuth redirect URI:
   - Google Cloud Console → APIs & Services → Credentials → нужный OAuth 2.0 Client ID
   - Добавить в Authorized redirect URIs: `https://simplebounty.wpmix.net/api/auth/google-dashboard-callback`
   - Сохранить

7. Финальная верификация: `curl https://simplebounty.wpmix.net` → 200

## Details

**Infrastructure topology:**

```
Client → 62.109.14.209 (SSL termination, nginx reverse proxy)
       → 95.217.227.164:80 (nginx VM104, proxy_pass по Host header)
       → localhost:8097 (simplebounty-web PM2 process)
```

**Files:**

- `/etc/nginx/sites-available/simplebounty.wpmix.net` (VM104) — создан Task 1; если отсутствует — создать по образцу simpledashboard с `proxy_pass http://127.0.0.1:8097`
- `/etc/nginx/sites-enabled/simplebounty.wpmix.net` (VM104) — симлинка, создать в этом таске
- `/etc/nginx/sites-available/simplebounty.wpmix.net` (62.109.14.209) — новый файл (reverse proxy vhost)
- `/etc/nginx/sites-enabled/simplebounty.wpmix.net` (62.109.14.209) — симлинка

**Шаблон nginx vhost на reverse proxy (62.109.14.209):**

```nginx
server {
    listen 80;
    server_name simplebounty.wpmix.net;

    location / {
        proxy_pass http://95.217.227.164;
        proxy_http_version 1.1;
        proxy_set_header Host simplebounty.wpmix.net;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300;
        proxy_send_timeout 300;
        proxy_connect_timeout 60;
    }
}
```

После certbot добавится listen 443 + SSL автоматически.

**PM2 процесс:**

`simplebounty-web` описан в `botplatform/ecosystem.config.js` (добавлен в Task 1). CWD: `/root/aisell/botplatform`. Script: `start-webchat-simplebounty.sh`. Port: 8097. Запускать из `/root/aisell/botplatform/`.

**Google OAuth (Decision 8):**

`GOOGLE_CLIENT_ID` задан в `start-webchat-simplebounty.sh`. Если тот же Client ID что у simpledashboard — просто добавить ещё один redirect URI в тот же OAuth app. Добавить: `https://simplebounty.wpmix.net/api/auth/google-dashboard-callback`.

**Cloudflare DNS:**

Скрипт `/root/space2/cloudflare_subdomain_manager.js` читает токен из `/root/space2/.env`. Добавлять с явным IP 62.109.14.209 (reverse proxy), не 95.217.227.164.

**Dependencies:**

- Task 1 — должен создать `start-webchat-simplebounty.sh`, запись `simplebounty-web` в `ecosystem.config.js`, и nginx vhost на VM104
- Task 12 — E2E тесты прошли (зависимость по depends_on)

**Edge cases:**

- Если certbot не находит домен — DNS ещё не применился; подождать 1-2 минуты и повторить
- Если `pm2 start --only simplebounty-web` выдаёт ошибку — убедиться что процесс описан в ecosystem.config.js (Task 1)
- Если порт 8097 занят — `lsof -i :8097`, завершить конфликтующий процесс
- Если nginx -t на VM104 выдаёт ошибку в sites-available/simplebounty — Task 1 не завершён; создать конфиг вручную
- Если Google Cloud Console не даёт добавить redirect URI — проверить, что редактируется правильный OAuth 2.0 Client ID (тот, чей GOOGLE_CLIENT_ID в start-webchat-simplebounty.sh)

**Rollback:**

- PM2: `pm2 stop simplebounty-web`
- DNS: `node /root/space2/cloudflare_subdomain_manager.js delete simplebounty.wpmix.net`
- Nginx VM104: `rm /etc/nginx/sites-enabled/simplebounty.wpmix.net && nginx -s reload`
- Nginx reverse proxy: удалить симлинку + файл + `nginx -s reload`
