## Details

**Files:**
- `/root/aisell/.gitignore` â€” append new "# Internal product files (not for public repo)" section with 4 pattern lines

**Why `git rm --cached` is required even though `.env` already exists in `.gitignore`:**
Git tracks files that were committed before a `.gitignore` rule existed. Once tracked, `.gitignore` rules have no effect on those files â€” the file must be explicitly removed from the index with `git rm --cached`. Running `git check-ignore -v bananzabot/.env` will confirm whether the existing `.env` rule matches it; either way, it remains tracked until `git rm --cached` is run.

**`git rm --cached` vs `git rm`:**
Always use `--cached`. This removes the file from the git index only â€” the file stays on disk. Running plain `git rm` deletes from disk, which would destroy live configuration.
