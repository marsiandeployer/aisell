## Description

Переименовываем `products/simple_site/CLAUDE.md.template` в `products/simple_site/SKILL.md` — для архитектурной консистентности со всеми остальными продуктами экосистемы.

`simple_site` — это AI-конструктор лендингов (Tailwind, Hero, i18n EN/RU, AI-фоны через Hydra). Текущий `CLAUDE.md.template` содержит рабочий набор инструкций, но он platform-specific: секция `## Домен` описывает бесплатный поддомен `u{USERID}.habab.ru` нашей webchat-платформы — это деталь только нашего хостинга, недопустимая в публичном skill-файле.

Задача решает две проблемы:
1. **Консистентность именования**: все продукты с шаблонами должны иметь `SKILL.md`, а не `CLAUDE.md.template`.
2. **Публичная безопасность**: из файла удаляются ссылки на платформенный домен (`u{USERID}.habab.ru`), которые бессмысленны для standalone skill-пользователей и не должны попасть в публичный репозиторий.

В этом спринте `simple_site` получает только переименование + минимальный YAML frontmatter + удаление платформенной секции. `marketplace.json` для `simple_site` — вне скопа, только `simple_dashboard` публикуется на маркетплейсе в этой фиче.

Webchat-поведение: `bot.ts` (Task 4) обновляет путь `getClaudeMdTemplatePath()` на `SKILL.md` — после переименования оба продукта (`simple_dashboard`, `simple_site`) начинают обслуживаться из нового имени файла. Поэтому переименование (Tasks 1+3, Wave 2) должно быть завершено до обновления `bot.ts` (Task 4, Wave 3).

## What to do

1. Прочитать текущий `products/simple_site/CLAUDE.md.template` — понять структуру и содержание.
2. Создать `products/simple_site/SKILL.md` с таким содержимым:
   - YAML frontmatter в начале файла: `name: simplesite`, `description: <непустая строка, описывающая назначение продукта>`, `version: "1.0.0"`, `tags: [site, landing, html, tailwind, spa]`.
   - Весь контент из `CLAUDE.md.template`, кроме секции `## Домен` — её удалить полностью (от заголовка `## Домен` до следующего заголовка `## Генерация сайтов`).
   - Секция `## Идея` с плейсхолдером `{{PROJECT_IDEA}}` — удалить (она webchat-specific, standalone skill-пользователи не проходят наш onboarding).
   - Секция `## Безопасность` — оставить (явное требование из user-spec).
3. Удалить `products/simple_site/CLAUDE.md.template`.
4. Убедиться, что в `SKILL.md` не осталось ни одного упоминания: `u{USERID}`, `localhost:`, `95.217.`, `62.109.`.

**Dependencies:**

- Task 1 (Create SKILL.md for simple_dashboard) — независима, выполняется параллельно (Wave 2)
- Task 4 (Update bot.ts template path) — зависит от Task 3. Bot.ts нельзя обновлять пока `SKILL.md` не создан: `getClaudeMdTemplatePath()` сразу начнёт возвращать новый путь, и если файл не существует — webchat упадёт с ENOENT.

**Edge cases:**

- `{{PROJECT_IDEA}}` присутствует в `CLAUDE.md.template` в секции `## Идея` — удалить вместе с секцией. В итоговом `SKILL.md` плейсхолдера быть не должно: `buildClaudeMdContent()` в bot.ts делает строковый replace только если маркер присутствует, но standalone skill-пользователи не проходят наш onboarding, где эта подстановка происходит.
- Секция `## Домен` содержит `u{USERID}.habab.ru` — специфика нашей webchat-платформы. Удаляется целиком. Standalone пользователь деплоит на GitHub Pages / Netlify / Vercel (это документируется в SKILL.md `simple_dashboard` через секцию `## Deployment`, для `simple_site` достаточно убрать платформенную секцию).
- Раздел `## AI-фоны` содержит Python-код с `HYDRA_API_KEY` env var — это общий паттерн с env vars, не хардкоженный ключ. Оставить как есть (env var ссылка безопасна).
- После удаления файла `CLAUDE.md.template` необходимо убедиться, что git видит изменение: `git status` должен показать удалённый файл и новый файл.

**Implementation hints:**

- Создать `SKILL.md` через Write tool, а не через `cp + edit` — так проще контролировать итоговое содержание.
- YAML frontmatter должен быть строго первым в файле (перед любым другим контентом).
- Для description в frontmatter: одна строка, описывающая назначение продукта для marketplace-индексации. Пример: "Build beautiful landing pages and websites as single-file index.html. Includes Tailwind CSS, bilingual EN/RU support, AI backgrounds."
- `tags` — строго массив YAML `[site, landing, html, tailwind, spa]`.
- Удалить `CLAUDE.md.template` через bash: `rm /root/aisell/products/simple_site/CLAUDE.md.template`.
