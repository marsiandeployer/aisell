# Task 5: CLAUDE.md.template auth section

## Required Skills

Before starting, load: `/skill:prompt-master` вАФ [SKILL.md](/root/.claude/skills/prompt-master/SKILL.md)

## Description

The SimpleDashboard product works by giving Claude a `CLAUDE.md` file in each user's workspace. Claude reads this file as its system prompt and uses it to know how to generate `index.html` dashboards. The template at `products/simple_dashboard/CLAUDE.md.template` is what gets copied into every new user's workspace when they onboard.

Currently the template teaches Claude: dashboard structure (SPA routing, i18n, Chart.js patterns), data source integration, industry templates, and output rules. It knows nothing about authentication.

This task adds an "Auth" section to the template so Claude can generate `index.html` files with complete Web3 auth code. The section must teach Claude:

1. **When to add auth** вАФ opt-in trigger conditions (user requests privacy, data is sensitive). Existing dashboards are not affected.
2. **What to embed** вАФ the `OWNER_ADDRESS` constant (a literal Ethereum address like `0x1234...abcd` passed by webchat when generating the dashboard).
3. **Auth flow code** вАФ the complete JavaScript pattern: ethers.js CDN, `window.ethereum` detection, challenge generation, fetch to `/api/auth/login`, JWT storage in `sessionStorage`, and reactive JWT refresh on 401 from any dashboard fetch.
4. **Overlay states** вАФ four distinct blur overlay states covering all failure modes:
   - No extension installed вЖТ blur + "Install SimpleDashboard extension" + Chrome Web Store link
   - Wrong address or Auth API returns 401 вЖТ blur + "No access to this dashboard"
   - Auth API down or fetch error вЖТ blur + "Authorization service temporarily unavailable"
   - Extension installed but keypair missing (error code `NO_KEYPAIR`) вЖТ blur + "Contact support@onout.org for access recovery"

The audience of this section is Claude (the AI agent), not end users. The prompt must be written for high-quality, consistent code generation across many dashboard sessions. Quality criterion: when an implementing agent generates a test dashboard with auth, the output HTML must contain correct, runnable auth code with the OWNER_ADDRESS embedded and all four overlay states present.

1. Read the existing `CLAUDE.md.template` to understand its style, tone, and code example format. The new section must fit naturally alongside existing sections.

2. Draft the "Auth" section content covering:
   - Trigger conditions: when Claude should add auth (user explicitly requests it, or data in the dashboard is described as private/sensitive/confidential). State that auth is opt-in and existing dashboards without auth are left unchanged.
   - The `OWNER_ADDRESS` constant: webchat passes it as a literal value like `0xAbc123...` in the conversation when auth is requested. Claude must embed it as a `const OWNER_ADDRESS = '0x...'` in the generated HTML.
   - Required CDN: `https://cdn.jsdelivr.net/npm/ethers@6/dist/ethers.min.js` вАФ add to `<head>` alongside existing Tailwind and Chart.js CDNs.
   - Auth init function: `initDashboardAuth()` called from `DOMContentLoaded`. The function checks `window.ethereum`, handles all four overlay states, and on success removes blur from the data container.
   - Challenge format: `JSON.stringify({ dashboardId: DASHBOARD_ID, timestamp: Date.now(), nonce: crypto.randomUUID() })` where `DASHBOARD_ID` is the subdomain numeric part (extract from `window.location.hostname`).
   - Login fetch: `POST /api/auth/login` with `{ signature, challenge, dashboardId }`, store JWT in `sessionStorage`.

   - Blur implementation: wrap chart/data content in a container with CSS `filter: blur(8px)` and `pointer-events: none`. Title and description remain unblurred (visible to unauthenticated visitors as a preview).
   - Overlay HTML and CSS pattern for all four states.

3. Include a self-contained code example in the section. The example must be complete enough that Claude can reproduce the pattern for any dashboard without additional lookup. Use the same format as other code examples in the template (fenced code blocks with comments).

4. Append the "Auth" section to `CLAUDE.md.template` after the existing "–Я—А–∞–≤–Є–ї–∞" / "Rules" section.
