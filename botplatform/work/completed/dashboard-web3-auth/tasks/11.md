## Details

### Secret generation

Generate and store secrets before starting PM2. File path must be `/root/aisell/botplatform/.env.auth`, mode 600. Example generation commands:

```bash
JWT_SECRET=$(openssl rand -hex 64)
INTERNAL_API_KEY=$(openssl rand -hex 32)
echo "JWT_SECRET=$JWT_SECRET" > /root/aisell/botplatform/.env.auth
echo "INTERNAL_API_KEY=$INTERNAL_API_KEY" >> /root/aisell/botplatform/.env.auth
chmod 600 /root/aisell/botplatform/.env.auth
```

Never commit `.env.auth` to git — add to `.gitignore` if not already there.

### PM2 start command

Start Auth API as a PM2 process using tsx. **Note:** PM2 CLI does NOT have a `--env` flag. Use shell `export` or `env` prefix to pass environment variables:

```bash
cd /root/aisell/botplatform
source .env.auth
AUTH_API_PORT=8095 \
PG_HOST=10.10.10.2 \
PG_PORT=5432 \
PG_DATABASE=dashboard_auth \
PG_USER=dashboard_auth \
JWT_SECRET=$JWT_SECRET \
INTERNAL_API_KEY=$INTERNAL_API_KEY \
pm2 start /root/aisell/botplatform/node_modules/.bin/tsx \
  --name dashboard-auth-api \
  -- src/auth-api.ts
pm2 save
```

Alternatively, the process is already configured in `ecosystem.config.js` (added by Task 2) with `env:` block containing all required variables. In that case, simply:
```bash
source .env.auth
pm2 start ecosystem.config.js --only dashboard-auth-api
pm2 save
```
The `source .env.auth` step is needed so that `JWT_SECRET` and `INTERNAL_API_KEY` are in the shell environment when PM2 reads the ecosystem config (the config references `process.env.*` for secrets).

### Nginx reload

The nginx location block for `/api/auth/` was added in Task 4. Reload only; no config changes needed:

```bash
sudo nginx -t && sudo systemctl reload nginx
```

If `nginx -t` fails, do NOT reload. Check `/etc/nginx/sites-enabled/d-wildcard.wpmix.net` for syntax errors introduced in Task 4.

### Extension rebuild and publish

Build for SimpleDashboard:

```bash
cd /root/aisell/extensions/webchat-sidebar
node build.js --name "SimpleDashboard" --url "https://simpledashboard.wpmix.net"
```

For Chrome Web Store publish — check if `REFRESH_TOKEN` is configured in `.env`. If yes, run `./scripts/publish-to-chrome-store.sh`. If not, send `out/webchat-sidebar.zip` to Telegram (see `chrome-extension-publishing.md` Step 6) for manual upload at: https://chrome.google.com/webstore/devconsole/hhdhmbcogahhehapnagdibghiedpnckn

Note: Chrome Web Store review for content scripts with `window.ethereum` injection may take 1-3 business days. The deploy task is complete once the extension is submitted, regardless of review completion.

### Simpledashboard webchat restart

The process is managed by PM2. If it exists already:

```bash
pm2 restart simpledashboard-web
```

If it was never started via PM2:

```bash
pm2 start bash --name simpledashboard-web -- /root/aisell/botplatform/start-webchat-simpledashboard.sh
pm2 save
```

### Rollback plan

| Step | Rollback action |
|------|-----------------|
| npm install broke dependencies | `git checkout package-lock.json && npm ci` |
| Auth API crash-loops | `pm2 stop dashboard-auth-api` — other services unaffected |
| Nginx reload broke dashboard routing | `sudo nginx -s reload` with previous working config, or `sudo systemctl restart nginx` |
| Extension publish caused issues | Old extension version remains active in Chrome Web Store until review completes |
| Simpledashboard webchat won't start | Check logs `timeout 30s pm2 logs simpledashboard-web --lines 100 --nostream`, fix env vars |

### Edge cases

- If `pg` connection fails at startup: Auth API logs connection error but may still start. Check `PG_HOST`, `PG_USER`, `PG_DATABASE` env vars match Task 1 setup.
- If `tsx` binary not found: check `node_modules/.bin/tsx` exists after `npm install`. Alternative: use `node --loader ts-node/esm`.
- If port 8095 already in use: `lsof -i :8095` to find the conflicting process.
- If PM2 process already exists with same name from a previous failed attempt: `pm2 delete dashboard-auth-api` before re-adding.
- Chrome Web Store submission may require updated screenshots or description if content script was flagged in previous reviews. Keep manifest permissions minimal (only `storage`, `activeTab`, content script match pattern `https://d*.wpmix.net/*`).
